## 1. Contexto y Oportunidad

## 2. Visión Estratégica

### 2.1 Qué es Apps del Sur (redefinición)

Apps del Sur **no es un producto** — es la fábrica de software de GYS Activa. Su modelo es la **customización en masa**: soluciones personalizadas a cada cliente pero escalables gracias al desarrollo asistido por IA.

- Primer vertical: gestión de mano de obra agrícola (viñas, frutícolas).
- Verticales futuros: servicios (funerarias, retail), industria, otros sectores.
- Modelo: cada implementación se adapta 100% a la operación del cliente.
- Diferenciador: el costo de customización baja drásticamente con IA, permitiendo software a medida a precios accesibles.

### 2.2 Cómo encaja Claude

La consultoría e implementación de Claude es un **servicio de Apps del Sur**, no un negocio separado. Son dos puertas de entrada al mismo edificio:

- **Puerta agro:** "Necesito un cuaderno de campo digital" → Apps del Sur construye la solución usando Claude.
- **Puerta IA:** "Necesito implementar IA en mi empresa" → Apps del Sur implementa Claude y construye las soluciones necesarias.

En ambos casos, GYS Activa es la empresa, Apps del Sur es la marca, y Claude es la herramienta principal.

### 2.3 Posicionamiento buscado

> "Apps del Sur es la fábrica de software del sur de Chile que construye soluciones a medida con Claude AI. Somos el referente de Claude en regiones — el partner que entiende tu operación porque vivimos donde tú vives."

## 3. Estrategia de Dominio y Marca

### 3.1 Decisión: un solo dominio

**appsdelsur.cl** será el único dominio activo. Razones:

1. Apps del Sur como fábrica de software multisector ya cubre ambos servicios.
2. Un solo dominio acumula toda la autoridad SEO.
3. Menos esfuerzo de mantenimiento con 5-10 hrs/semana disponibles.
4. Ya tiene blog con 9 artículos y SEO funcionando.
5. Enfoque lean: no gastar recursos en segundo sitio hasta validar.

### 3.2 Propiedad intelectual — por qué no usar "Claude" en el dominio

- "CLAUDE" es marca registrada de Anthropic, PBC (USPTO).
- Los términos de Anthropic prohíben sugerir partnership, patrocinio o endorsement sin aprobación escrita.
- La estrategia correcta: usar "Claude" como keyword de posicionamiento en contenido (titles, H1, meta descriptions, blog posts), no en el dominio.
- Al ser Claude Partner oficial, se puede usar el badge de partner según sus guidelines.

### 3.3 Cómo posicionar "Claude" sin tenerlo en el dominio

| Elemento | Ejemplo |

| Title tag home | "Apps del Sur — Software a medida con IA para empresas de Chile" |

## 5. Certificación Claude Certified Architect (CCA) Foundations

### 5.1 Requisitos

- Membresía en Claude Partner Network (gratis, aplicar como GYS Activa).
- Se recomienda 6+ meses de experiencia con Claude API y Claude Code.
- Examen: 60 preguntas multiple choice, 120 minutos, puntaje mínimo 720/1000.
- Proctored, basado en escenarios reales de producción.

### 5.2 Dominios del examen

| Dominio | Peso | Descripción |
|---------|------|-------------|
| Agentic Architecture & Orchestration | 27% | Sistemas multi-agente, task decomposition, hub-and-spoke |
| Claude Code Configuration & Workflows | 20% | CLAUDE.md, skills, MCP servers, plan mode |
| Prompt Engineering & Structured Output | 20% | Few-shot, JSON schemas, validation loops |
| Tool Design & MCP Integration | 18% | Diseño de herramientas, MCP servers, error handling |
| Context Management & Reliability | 15% | Contexto largo, escalación, error propagation |

### 5.3 Recursos de preparación (gratuitos)

- **Anthropic Academy en Skilljar** (anthropic.skilljar.com): 13 cursos gratuitos.
  - Claude 101 (fundamentos)
  - Building Applications with the Claude API (curso core, 8.1 horas)
  - Model Context Protocol (MCP) básico y avanzado
  - AI Fluency Framework
- **Coursera**: "Building with the Claude API" y "Real-World AI for Everyone" (Anthropic + AWIT).
- **Práctica**: Construir proyectos reales con Claude API y Claude Code (Apps del Sur es tu lab).

### 5.4 Plan de preparación

## 9. Criterios de Decisión Lean

### ¿Cuándo saber si enfocarse más en agro o en consultoría Claude?

| Señal | Interpretación | Acción |
|-------|---------------|--------|
| Primer cliente pagado es agro | El mercado validó el vertical agro primero | Mantener Claude como servicio secundario, seguir publicando contenido |
| Primer cliente pagado es consultoría Claude (no agro) | El mercado validó la consultoría IA | Evaluar si crear página dedicada / marca separada. El contenido ya existe |
| Ambos traccionan | GYS Activa es un holding de servicios tech | Apps del Sur = fábrica + consultoría integrada. Potenciar ambos |
| Ninguno tracciona en 4 meses | Revisar propuesta de valor y canales | Pivotar enfoque, revisar pricing, explorar otros canales |

### ¿Cuándo separar en dos marcas/dominios?

Solo si se cumplen las tres condiciones simultáneamente:
1. Hay clientes Claude que no son agro (audiencia diferente confirmada).
2. El tráfico a /claude justifica un sitio propio (métricas reales).
3. Hay tiempo y recursos para mantener dos sitios con calidad.

Hasta entonces, un solo dominio es la respuesta correcta.
